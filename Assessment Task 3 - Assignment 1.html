Name Web Attacks
Broken Access Control

Source: Broken Access Control 		What is it?
Broken Access Control is a security vulnerability that allows users to gain unauthorised access to areas of an application that they were not meant to have access to.
How can it occur?
Broken Access Control can occur due to flaws in the design or implementation of access control mechanisms, whereby access is granted to unintended users.
How to prevent it?
To prevent Broken Access Control, implement and thoroughly test the access control mechanisms that have been put in place before deploying the application.

Source: https://owasp.org/Top10/A01_2021-Broken_Access_Control/
Cryptographic Failures

Source: Cryptographic Failures 		What is it?
Cryptographic Failures is a security vulnerability where there is a failure in the use of cryptographic algorithms.
How can it occur?
Cryptographic Failures can occur due to failures in the design or implementation of cryptographic algorithms or due to using weak cryptographic algorithms.
How to prevent it?
To prevent Cryptographic Failures, use up-to-date and strong cryptographic algorithms and random, non-default keys.

Source: https://owasp.org/Top10/A02_2021-Cryptographic_Failures/
Injection

Source: Injection 		What is it?
Injection is a security vulnerability where untrusted data is sent to the application from the user, such as via a user input field.
How can it occur?
Injection can occur when input data from the user is not sanitised or validated before being transmitted to the application.
How to prevent it?
To prevent Injection, user input data should be sanitised and validated before being transmitted to the application.

Source: https://owasp.org/Top10/A03_2021-Injection/
Insecure Design

Source: Insecure Design 		What is it?
Insecure Design is a security vulnerability whereby the application has design and architectural flaws.
How can it occur?
Insecure Design can occur when security is not considered during the design and architectural phases of the application's development.
How to prevent it?
To prevent Insecure Design, security should be considered during all phases of the application's development lifecycle.

Source: https://owasp.org/Top10/A04_2021-Insecure_Design/
Security Misconfiguration

Source: Security Misconfiguration 		What is it?
Security Misconfiguration is a security vulnerability whereby an application has not been configured with the appropriate parameters.
How can it occur?
Security Misconfiguration can occur when an application has not been properly hardened during configuration.
How to prevent it?
To prevent Security Misconfiguration, ensure that the application has been properly hardened with the appropriate security controls during configuration.

Source: https://owasp.org/Top10/A05_2021-Security_Misconfiguration/
Vulnerable and Outdated Components

Source: Vulnerable and Outdated Components 		What is it?
Vulnerable and Outdated Components is a security vulnerability in which the application uses components that are not up-to-date.
How can it occur?
Vulnerable and Outdated Components can occur when the application uses components that are not up-to-date.
How to prevent it?
To prevent Vulnerable and Outdated Components, ensure that all components used in the application are up-to-date.

Source: https://owasp.org/Top10/A06_2021-Vulnerable_and_Outdated_Components/
Identification and Authentication Failures

Source: Identification and Authentication Failures 		What is it?
Identification and Authentication Failures is a security vulnerability when a user is not properly identified and authenticated as another user.
How can it occur?
Identification and Authentication Failures can occur when a user is identified as another user and granted access to the incorrect user's account.
How to prevent it?
To prevent Identification and Authentication Failures, multi-factor authentication can be implemented.

Source: https://owasp.org/Top10/A07_2021-Identification_and_Authentication_Failures/
Software and Data Integrity Failures

Source: Software and Data Integrity Failures 		What is it?
Software and Data Integrity Failures is a security vulnerability when an application's infrastructure and code do not protect against security violations.
How can it occur?
Software and Data Integrity Failures can occur when the received data is not validated and verified.
How to prevent it?
To prevent Software and Data Integrity Failures, data should be validated and verified using mechanisms such as checksums and digital signatures.

Source: https://owasp.org/Top10/A08_2021-Software_and_Data_Integrity_Failures/
Security Logging and Monitoring Failures

Source: Security Logging and Monitoring Failures 		What is it?
Security Logging and Monitoring Failures is a security vulnerability that allows security breaches to be missed due to insufficient logging and monitoring.
How can it occur?
Security Logging and Monitoring Failures can occur when there is insufficient logging and monitoring for security-related events.
How to prevent it?
To prevent Security Logging and Monitoring Failures, ensure that all security-related events are logged and monitored.

Source: https://owasp.org/Top10/A09_2021-Security_Logging_and_Monitoring_Failures/
Server-Side Request Forgery

Source: Server Side Request Forgery 		What is it?
Server-Side Request Forgery is a security vulnerability where an application does not validate a user-supplied URL when fetching a remote resource.
How can it occur?
Server-Side Request Forgery can occur when an application does not validate a user-supplied URL when fetching a remote resource.
How to prevent it?
To prevent Server-Side Request Forgery, ensure that all user-supplied URLs are validated before fetching a remote resource.

Source: https://owasp.org/Top10/A10_2021-Server-Side_Request_Forgery_%28SSRF%29/
